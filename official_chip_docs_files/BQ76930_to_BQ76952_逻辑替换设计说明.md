# BQ76930 到 BQ76952 逻辑替换设计说明

记录日期：2026-06-25  
适用阶段：Phase 1 bring-up / 旧项目风格平移  
目标芯片：BQ76952PFBR  
旧项目芯片：BQ7693003DBTR  
电芯：EVE-INR21700/50E，第一版按 6S1P 推导

---

## 1. 文档目标与边界

本文档用于指导旧 UPS 项目中围绕 `App_BatMan` / BQ76930 的业务逻辑，迁移到新 BQ76952 硬件环境。

本文件不是“重新设计整套 BMS 软件架构”的长期量产终态文档，而是 **Phase 1 bring-up / 教学型平移替换方案**。核心目标是：

- 旧项目中清晰的 `App_BatMan` 步骤式业务流程可以继承。
- 旧项目 BQ76930 的寄存器地址、ADC 换算公式、保护寄存器公式、均衡寄存器写法不能继承。
- 新项目第一阶段允许 APP 直接调用 `Int_BQ76952_*` 协议原语，便于上板、示波、串口打印和教学。
- COM 层保持旧项目风格：只做查表、纯计算、无副作用辅助，不重新变成 BQ 厚语义层。
- INT 层保留 BQ76952 必需的 direct command、subcommand、Data Memory、CRC、transfer buffer checksum 访问模型。
- 所有 BQ76952 Data Memory 最终写入值、FET status bit 解析、保护阈值编码、NTC 配置编码必须以 TRM、BSP 宏和实测为准，禁止拍脑袋写死。

后续若硬件、电流校准、温度采样、FET 读回和保护参数稳定，可以进入 Phase 2，把部分芯片语义重新收敛到 `COM/BQIF` 层。

## 3. 替换总原则

### 3.1 可继承内容

| 旧项目内容 | 继承方式 |
|---|---|
| `App_BatMan` 步骤式初始化 | 保留为新 `App_BatMan_Init()` 主流程 |
| 单体电压、总压、电流、温度分别独立采样 | 保留函数粒度，但内部改为 BQ76952 direct command |
| SOC 调用 `App_Soc_Update()` + 显示滤波 | 保留结构，容量和电流方向重新校准 |
| 均衡策略：最低电压、阈值筛选、相邻互斥 | 保留业务策略，最终写 BQ76952 16-bit host balance mask |
| `printf` 密集调试 | Phase 1 保留，便于 bring-up |
| COM 查表风格 | 保留，但只做纯计算/查表，不访问 BQ |

### 3.2 禁止继承内容

| 旧项目内容 | 禁止原因 |
|---|---|
| BQ76930 寄存器地址 | BQ76952 访问模型完全不同 |
| `BQ_SYS_CTRL2.CHG_ON / DSG_ON` | BQ76952 FET 控制方式不同 |
| `BQ_CELLBAL1/BQ_CELLBAL2` | BQ76952 使用 `CB_ACTIVE_CELLS` 16-bit mask |
| `cell_reg_indexes[6] = {0,1,4,5,6,9}` | 新硬件为 Cell1/2/6/9/12/16，旧映射不适用 |
| 旧 ADC gain/offset 电压公式 | BQ76952 direct command 直接给 mV/userV |
| 旧 CC 电流公式 `cc_adc * 8.44 / 1000 / 5.0` | 新采样电阻与 BQ 计量体系不同 |
| 旧 TS1 热敏公式 | 新温度资源分布不同，TS2 还被复用为 WAKE |
| `BQ769_RegisterGroup` 全局位域镜像 | 会把 BQ76930 思维带到新芯片 |
| 旧 OV/UV trip 公式 | BQ76952 Data Memory 单位/编码不同 |

## 4. 旧项目 BQ76930 软硬件事实

### 4.1 旧硬件事实

旧控制板 BQ 页面使用 `BQ7693003DBTR`。围绕 BQ76930 的主要信号包括：

- `DSG`
- `CHG`
- `SDA`
- `SCL`
- `TS1`
- `VC0..VC10`
- `SRP/SRN`
- `ALERT`

旧电池板是 6 节 14500/AA 结构，电芯网络为：

```text
CELL0, CELL1, CELL2, CELL3, CELL4, CELL5, CELL6
```

旧项目软件通过 `cell_reg_indexes[6] = {0, 1, 4, 5, 6, 9}` 把 6 个物理电芯映射到 BQ76930 的 VC 寄存器序列。这是旧硬件与 BQ76930 的历史映射，不能迁移到 BQ76952。

### 4.2 旧 INT 层

旧 `Int_BQ769` 是薄硬件访问层：

| 方法 | 旧项目职责 | 迁移判断 |
|---|---|---|
| `Int_BQ769_WakeUp()` | GPIO 唤醒 | 仅继承“板级唤醒需要独立函数”的思想 |
| `Int_BQ769_Ship()` | 写 `SYS_CTRL1.SHUT_A/SHUT_B` 进 ship | 不迁移旧寄存器序列 |
| `Int_BQ769_Reset()` | `Ship()` + `WakeUp()` | 新项目需区分 BQ subcommand reset 与 `RST_SHUT` 板级时序 |
| `Int_BQ769_WriteReg()` | 单字节寄存器写 + CRC | 不适配 BQ76952 direct/subcommand/Data Memory |
| `Int_BQ769_ReadReg()` | 连续寄存器读 + 每字节 CRC | 仅继承 CRC 校验严谨性 |

旧 INT 使用 I2C 地址 `0x10/0x11`、`hi2c2`、FreeRTOS 临界区、CRC8 多项式 `0x07`。读写失败多为 `printf`，没有系统状态码。这一风格适合 bring-up，但新项目应至少保留 `comm_ok/error_flags`。

### 4.3 旧 COM 层

旧 `Com_Bq769` 不访问 BQ 芯片，只做查表：

| 内容 | 旧职责 | 新项目处理 |
|---|---|---|
| `resist_temper_table[181]` | NTC 电阻转温度 | 等新 NTC 参数确认后重建 |
| `voltage_soc_ocv_table[101]` | 单体 OCV-SOC 表 | 第一版可复用表结构，但必须标注待 EVE 50E 曲线校准 |
| `getTableIndexByValue()` | 查表索引 | 可继承 |
| `Com_BQ769_getTemperByResist()` | 电阻转温度 | 参数重做后可继承接口风格 |
| `Com_BQ769_getPercentByVoltage()` | 电压转 SOC | 可继承接口风格 |

### 4.4 旧 APP 层

旧 `App_BatMan` 是 BQ 业务主文件：

| 方法 | 旧逻辑 | 新项目替换方向 |
|---|---|---|
| `App_BatMan_Init()` | reset/wake、读 gain/offset、配置 ADC/CC/FET、配置保护、读回、SOC init | 改为 BQ76952 DeviceNumber、ConfigUpdate、Vcell Mode、DA/FET/CB 配置、读回 |
| `App_BatMan_loadCellsVoltage()` | 读 `BQ_VC1_HI + index*2`，旧公式换算 | 读 BQ76952 6 个 direct command，直接 mV |
| `App_BatMan_loadBatVoltage()` | 读 `BQ_BAT_HI`，旧公式换算 | 读 `Stack Voltage` / `PACK Pin Voltage` / `LD Pin Voltage` |
| `App_BatMan_loadCurrent()` | 读 `BQ_CC_HI`，旧 shunt 公式 | 读 `CC2 Current(0x3A)`，单位与方向实测确认 |
| `App_BatMan_loadTemperature()` | 读 TS1 ADC，旧 10k 分压公式 | 第一版读 `Int Temperature`，后续配置 TS/DCHG/DFETOFF |
| `App_BatMan_CalcSoc()` | `App_Soc_Update(interval_ms, bat_mv, current_a)` + display filter | 保留，但容量、电流方向、OCV 表重做 |
| `App_BatMan_CellBalance()` | 最低电压、候选筛选、相邻互斥、写 `CELLBAL1/2` | 保留策略，写 `CB_ACTIVE_CELLS(0x0083)` |
| `SetChargeState/SetDischargeState()` | 读改写 `SYS_CTRL2.CHG_ON/DSG_ON` | 用 BQ76952 FET subcommand 或 `FET_CONTROL(0x0097)`，并读回 `FET Status(0x7F)` |

### 4.5 旧 SOC 参数必须替换

旧 `App_Soc` 中容量为 UPS 旧电池容量，例如 `capacity_ah = 0.6f` 或等价库仑参数。这必须替换为 EVE 50E pack 容量：

- 6S1P 典型容量：`5.0Ah`
- 6S1P 保守容量：`4.9Ah`
- 若实物为 6SxP：容量按 P 倍放大

旧 `App_Soc.h` 语义中 `current_a > 0` 表示充电。新 BQ76952 `CC2 Current` 的符号必须上板确认后再接入 SOC。

## 5. 新项目硬件事实

### 5.1 新 BMS 板总体

新 BMS 板标题为 `6S 三元锂 高输出能力BMS`，核心为 `BQ76952PFBR`。

| 项目 | 硬件事实 |
|---|---|
| BQ 芯片 | `BQ76952PFBR` |
| 串数 | 6S，稀疏映射到 BQ Cell1/2/6/9/12/16 |
| 电芯接口 | `CN2` 1x7，`CELL_6+` 到 `CELL_1-` |
| 功率接口 | XT90，`BAT+ / BAT-`，输出 `BMS_OUT+ / BMS_OUT-` |
| 低边采样 | `R18=0.5mΩ ±1% 6W`，一端 GND，一端 `BMS_OUT-` |
| 均衡电阻 | `R3/R4/R5/R6/R7/R19=62Ω ±1% 1W` |
| I2C | `I2C1`，`PB6=SCL`，`PB7=SDA` |
| ALERT | `PB4 = BMS_INT/ALERT`，控制板侧 10k 上拉 SYS_3V3 |
| WAKE | `BMS_WAKE` 接 BQ `TS2`，关机拉低唤醒，运行时复用按键检测 |
| SHUT | `BMS_CHIP_SHUT` 接 BQ `RST_SHUT`，时序待实测确认 |
| ONLINE | `BMS_CHIP_ONLINE` 接 BQ `REG18`，只能表示 BQ 内部 1.8V/在线电平，不表示软件初始化成功 |

### 5.2 BQ76952 U9 关键引脚连接

| BQ 引脚 | 网表/PDF 连接 | 软件含义 |
|---|---|---|
| `VC16` | 经 R8 100Ω 到 `CELL_6+` | 物理 Cell6 顶端，电压 direct command Cell16 |
| `VC12` | 经 R1 100Ω 到 `CELL_5+`，VC13/14/15 与其短接组 | 物理 Cell5 顶端，direct command Cell12 |
| `VC9` | 经 R9 100Ω 到 `CELL_4+`，VC10/11 与其短接组 | 物理 Cell4 顶端，direct command Cell9 |
| `VC6` | 经 R10 100Ω 到 `CELL_3+`，VC7/8 与其短接组 | 物理 Cell3 顶端，direct command Cell6 |
| `VC2` | 经 R11 100Ω 到 `CELL_2+`，VC3/4/5 与其短接组 | 物理 Cell2 顶端，direct command Cell2 |
| `VC1` | 经 R12 100Ω 到 `CELL_1+` | 物理 Cell1 顶端，direct command Cell1 |
| `VC0` | 经 R13 100Ω 到 `CELL_1-` | 电芯底端 |
| `SRP/SRN` | 经 100Ω + RC 到 0.5mΩ 采样电阻两端 | BQ 电流采样，符号需实测 |
| `TS1` | CN4 + C17 2.2nF | 外接温度资源，NTC 安装/参数待确认 |
| `TS2` | `BMS_WAKE` | 唤醒脚，不作为普通温度 |
| `TS3` | CN3 + C15 2.2nF | 外接温度资源，NTC 安装/参数待确认 |
| `DFETOFF` | R23 10k/B3950 NTC + C16 | PDF 标注复用为采样电阻温度检测 |
| `DCHG` | R22 10k/B3950 NTC + C14 | 可配置温度/ADC 输入，建议用于热监测 |
| `PACK/LD` | 10k 到 `BMS_OUT+` | pack/load 相关电压检测 |
| `DSG/CHG` | 经 100Ω 到功率 FET gate 驱动 | BQ 保护/FET 控制输出 |
| `PDSG` | `PDSG` 网 | 预放电/预充相关，第一版可暂不展开 |
| `RST_SHUT` | `BMS_CHIP_SHUT` + RC | 复位/关断，需确认短/长时序 |
| `REG18` | `BMS_CHIP_ONLINE` | BQ 在线供电信号 |
| `SCL/SDA` | 经 100Ω 到 MCU I2C1 | BQ 通信 |
| `ALERT` | `MUC_EXTI4_PB4_BMS_INT` | 中断/告警 |

### 5.3 新控制板与 BMS 接口

新控制板通过 1x7 接口连接 BMS：

| BMS 接口 pin | 信号 | 控制板连接 | 软件注意 |
|---|---|---|---|
| 1 | `BMS_CHIP_ONLINE` | MCU/检测电路 | 只表示 BQ REG18/在线电平，不能作为 init ok |
| 2 | `BMS_WAKE` | 按键/唤醒路径 | 原理图说明关机拉低 WAKE 唤醒，需实测 |
| 3 | `MUC_EXTI4_PB4_BMS_INT` | MCU PB4，10k 上拉 | ALERT 中断输入 |
| 4 | `MCU_I2C1_SCL_PB6_BMS` | MCU PB6，2k 上拉 SYS_3V3 | I2C1 SCL |
| 5 | `MCU_I2C1_SDA_PB7_BMS` | MCU PB7，2k 上拉 SYS_3V3 | I2C1 SDA |
| 6 | `BMS_CHIP_SHUT` | SW3/MCU 路径 | RST_SHUT 复位/关断时序待确认 |
| 7 | `BAT+` | 电池正端 | 功率/检测输入 |

### 5.4 SC8815 充电路径边界

新控制板上 `U4=SC8815` 是充电路径控制器，负责 24V 输入到电池侧的充电调节。它不替代 BQ76952 的电芯电压保护、均衡、FET 安全关断。

关键硬件事实：

- 输入侧采样：`R5=10mΩ/3W`
- 电池侧采样：`R14=10mΩ/3W`
- SC8815 I2C：`MCU_I2C3_SCL_PA7_SC8815` / `MCU_I2C3_SDA_PA6_SC8815`
- SC8815 控制：`#CE`、`PSTOP`、`INT`
- `VBATS` 分压风险：网表中 `R17=0Ω` 从 `VBATS` 到 `BMS+`，`R18=0Ω` 从 `GND` 到 `VBATS`，bring-up 前必须确认实物改焊/DNP，否则 SC8815 电池电压检测可能异常。

SC8815 只负责“怎么充”，BQ76952 负责“能不能安全充/放”。两者关系：

```text
APP 充电策略 -> SC8815 设置电压/电流
APP 安全策略 -> BQ76952 FET/保护状态
BQ76952 保护触发 -> 允许强制关断 CHG/DSG FET
```

### 6.3 SC8815 目标充电电压修正

原计划中 SC8815 目标充电电压约 `24.73V`。

```text
24.73V / 6 = 4.1217V/cell
```

该值高于 EVE 50E 在 `5A` 最大持续充电条件下的 `4.10V/cell`。因此：

| 目标 | 建议 |
|---|---|
| 坚持 `5A` 充电 | SC8815 充电目标应重新评估到约 `24.6V` |
| 坚持 `24.73V` | 应降低充电电流目标，或单独评估热、寿命和规格符合性 |
| 追求 `25.2V` 满充 | 应按 `1A` 标准充电更符合规格书 |

### 6.4 温度策略目标

| 温度范围 | 充电策略 |
|---|---|
| `< 0℃` | 禁止充电 |
| `0~15℃` | 充电限流到 `<=1A` |
| `15~45℃` | 允许最高 `5A` 充电 |
| `45~55℃` | 充电限流到 `<=1A` |
| `>55℃` | 禁止充电 |

| 温度范围 | 放电策略 |
|---|---|
| `< -20℃` | 禁止放电 |
| `-20~60℃` | 允许放电 |
| `>60℃` | 禁止放电或降级保护 |
| `>=70℃` | cell surface cut-off 硬边界 |

这些是电芯温度策略目标，不等于 BQ76952 TS/DCHG/DFETOFF Data Memory 的最终编码。

## 7. BQ76952 官方访问模型摘要

### 7.1 I2C 与 CRC

新项目 BQ 地址：

| 类型 | 地址 |
|---|---:|
| 7-bit | `0x08` |
| 8-bit write | `0x10` |
| 8-bit read | `0x11` |

默认 I2C CRC 开启。BQ76952 CRC 与旧 BQ76930 同为 CRC8 多项式 `0x07`，但 BQ76952 direct/subcommand 读写帧和 CRC 计算范围不同，必须按 TI Software Development Guide 实现。

要点：

- 写：首字节 CRC 覆盖地址、命令、数据；后续数据字节 CRC 通常按单字节计算。
- direct command 读：首个数据字节 CRC 覆盖 write address、command、read address、first data；后续字节 CRC 按数据字节计算。
- Data Memory 和 data subcommand 还要处理 transfer buffer checksum 和 length。

### 7.2 Direct Command

| Command | 名称 | 单位 | 本项目用途 |
|---:|---|---|---|
| `0x14` | Cell 1 Voltage | mV, I2 | 物理 Cell1 |
| `0x16` | Cell 2 Voltage | mV, I2 | 物理 Cell2 |
| `0x1E` | Cell 6 Voltage | mV, I2 | 物理 Cell3 |
| `0x24` | Cell 9 Voltage | mV, I2 | 物理 Cell4 |
| `0x2A` | Cell 12 Voltage | mV, I2 | 物理 Cell5 |
| `0x32` | Cell 16 Voltage | mV, I2 | 物理 Cell6 |
| `0x34` | Stack Voltage | userV, I2 | 电芯栈总压 |
| `0x36` | PACK Pin Voltage | userV, I2 | PACK/BMS_OUT+ 相关电压 |
| `0x38` | LD Pin Voltage | userV, I2 | 负载检测 |
| `0x3A` | CC2 Current | userA, I2 | 电流，方向/校准待实测 |
| `0x68` | Int Temperature | 0.1K, I2 | 第一版温度读数 |
| `0x70` | TS1 Temperature | 0.1K 或 mV | 视 pin 配置 |
| `0x74` | TS3 Temperature | 0.1K 或 mV | 视 pin 配置 |
| `0x78` | DCHG Temperature | 0.1K 或 mV | R22 NTC 资源 |
| `0x6C` | DFETOFF Temperature | 0.1K 或 mV | R23 NTC 资源 |
| `0x7F` | FET Status | H1 | FET 实际状态读回 |

### 7.3 Subcommand 与 Data Memory

BQ76952 通过 `0x3E/0x3F` 间接访问 subcommand。返回数据从 `0x40` transfer buffer 读取。Data Memory 写入或 data subcommand 需要写 checksum / length 到 `0x60/0x61`。

常用 subcommand：

| Subcommand | 名称 | 用途 |
|---:|---|---|
| `0x0090` | `SET_CFGUPDATE` | 进入配置更新模式 |
| `0x0092` | `EXIT_CFGUPDATE` | 退出配置更新模式 |
| `0x0093` | `DSG_PDSG_OFF` | 请求关闭 DSG/PDSG |
| `0x0094` | `CHG_PCHG_OFF` | 请求关闭 CHG/PCHG |
| `0x0095` | `ALL_FETS_OFF` | 请求关闭所有 FET |
| `0x0096` | `ALL_FETS_ON` | 请求允许所有 FET |
| `0x0097` | `FET_CONTROL` | data subcommand，按 bit 请求屏蔽 FET |
| `0x0083` | `CB_ACTIVE_CELLS` | host-controlled balancing，16-bit mask |
| `0x0085` | `CBSTATUS1` | 均衡状态/时间读回 |
| `0x0086` | `CBSTATUS2` | cell1-8 累计均衡状态 |
| `0x0087` | `CBSTATUS3` | cell9-16 累计均衡状态 |

### 7.4 关键 Data Memory 字段

以下字段可作为计划书和 BSP 宏核对清单。最终地址和值必须以项目 `Int_BQ76952_BSP.h` 与 TI TRM 复核为准。

| 地址 | 字段 | 用途 |
|---:|---|---|
| `0x9304` | `Settings:Configuration:Vcell Mode` | 连接电芯 bit mask，本项目应写 `0x8923` |
| `0x9303` | `DA Configuration` | 电压/电流测量相关配置 |
| `0x9308` | `FET Options` | FET 控制选项，host FET 控制相关 |
| `0x926D` | `Default Alarm Mask` | 默认告警输出屏蔽 |
| `0x9261/0x9262/0x9263` | `Enabled Protections A/B/C` | 保护使能 |
| `0x9265..0x926B` | `CHG/DSG FET Protections` | 保护关联 FET 动作 |
| `0x9275` | `CUV Threshold` | CUV 阈值，单位约 50.6mV |
| `0x9276` | `CUV Delay` | CUV 延时，单位约 3.3ms |
| `0x927B` | `CUV Recovery Hysteresis` | CUV 恢复滞回 |
| `0x9278` | `COV Threshold` | COV 阈值，单位约 50.6mV |
| `0x9279` | `COV Delay` | COV 延时，单位约 3.3ms |
| `0x927C` | `COV Recovery Hysteresis` | COV 恢复滞回 |
| `0x9280` | `OCC Threshold` | 充电过流，sense voltage 单位约 2mV |
| `0x9281` | `OCC Delay` | OCC 延时，单位约 3.3ms |
| `0x9282` | `OCD1 Threshold` | 放电过流 1，sense voltage 单位约 2mV |
| `0x9283` | `OCD1 Delay` | OCD1 延时 |
| `0x9284` | `OCD2 Threshold` | 放电过流 2 |
| `0x9285` | `OCD2 Delay` | OCD2 延时 |
| `0x9286` | `SCD Threshold` | 短路阈值档位 |
| `0x9287` | `SCD Delay` | 短路延时，快速保护 |
| `0x9335` | `Balancing Configuration` | 均衡配置，注意 `CB_NO_CMD` |
| `0x9336` | `Cell Balance Min Cell Temp` | 均衡最低温度 |
| `0x9337` | `Cell Balance Max Cell Temp` | 均衡最高温度 |
| `0x9338` | `Cell Balance Max Internal Temp` | 内部温度限制 |
| `0x933A` | `Cell Balance Max Cells` | 自动/级别均衡最大 cell 数 |
| `0x933B` | `Min Cell V (Charge)` | 充电均衡最低电压 |
| `0x933D` | `Min Delta (Charge)` | 充电均衡启动压差 |
| `0x933E` | `Stop Delta (Charge)` | 充电均衡停止压差 |

## 8. 新 6S Cell 映射与 `Vcell Mode`

### 8.1 物理到 BQ 通道映射

| 物理电芯 | 物理电压定义 | BQ 通道 | Direct Command | 均衡 bit | mask |
|---|---|---|---:|---:|---:|
| Cell1 | `CELL_1+ - CELL_1-` | Cell1 | `0x14` | bit0 | `0x0001` |
| Cell2 | `CELL_2+ - CELL_1+` | Cell2 | `0x16` | bit1 | `0x0002` |
| Cell3 | `CELL_3+ - CELL_2+` | Cell6 | `0x1E` | bit5 | `0x0020` |
| Cell4 | `CELL_4+ - CELL_3+` | Cell9 | `0x24` | bit8 | `0x0100` |
| Cell5 | `CELL_5+ - CELL_4+` | Cell12 | `0x2A` | bit11 | `0x0800` |
| Cell6 | `CELL_6+ - CELL_5+` | Cell16 | `0x32` | bit15 | `0x8000` |

### 8.2 `Vcell Mode` 计算

```text
Cell1  -> bit0  = 0x0001
Cell2  -> bit1  = 0x0002
Cell6  -> bit5  = 0x0020
Cell9  -> bit8  = 0x0100
Cell12 -> bit11 = 0x0800
Cell16 -> bit15 = 0x8000
--------------------------------
Vcell Mode = 0x8923
```

`Vcell Mode` 写入地址为 `0x9304`。写入后必须读回，并用万用表逐串验证 6 个 direct command 与物理电芯一致。

## 11. 初始化逻辑替换

### 11.1 目标流程

`App_BatMan_Init()` 建议写成旧项目风格的步骤式流程：

```text
1. 打印 BQ76952 init start
2. 板级唤醒：必要时控制 BMS_WAKE / 检查 BMS_CHIP_ONLINE
3. 读取 DeviceNumber / FW version，确认 I2C + CRC 通信
4. 可选：发送 RESET subcommand 或仅做 soft init
5. 进入 ConfigUpdate
6. 写 Vcell Mode = 0x8923
7. 写 DA Configuration / FET Options / Default Alarm Mask / Balancing Configuration 等已确认项
8. 写 Cell Balancing 相关保守参数
9. 暂不写死 COV/CUV/OCC/OCD/SCD 最终值，除非已完成 TRM 编码和实测复核
10. 读回关键 Data Memory 项并打印
11. 退出 ConfigUpdate
12. 清除 latched alarm/fault（按 TRM）
13. 读取 direct command：6 串电压、stack、pack、电流、温度
14. 根据策略请求 FET 允许状态，并读取 FET Status
15. 初始化 App_Soc
16. 打印 BQ76952 init result
```

### 11.2 ConfigUpdate 写入顺序建议

| 步骤 | 内容 | 说明 |
|---|---|---|
| 1 | `SET_CFGUPDATE(0x0090)` | 进入配置模式，必须确认状态 |
| 2 | 写 `Vcell Mode(0x9304)=0x8923` | 6S 稀疏映射核心配置 |
| 3 | 写 `DA Configuration` | 只写已确认值 |
| 4 | 写 `FET Options` | 若使用 host FET control，需确认相关 bit |
| 5 | 写 `Balancing Configuration` | 确保 host command 不被 `CB_NO_CMD` 禁止 |
| 6 | 写 `Cell Balance Max Cells=1` | 即使 TRM 允许更多，APP 第一版也只开一节 |
| 7 | 读回以上项目 | 读回不一致则置 `bq_config_ok=0` |
| 8 | `EXIT_CFGUPDATE(0x0092)` | 退出配置模式，确认状态 |

### 11.3 初始化错误处理

最低要求：

```c
#define APP_BATMAN_ERR_I2C          (1u << 0)
#define APP_BATMAN_ERR_CRC          (1u << 1)
#define APP_BATMAN_ERR_READBACK     (1u << 2)
#define APP_BATMAN_ERR_CFGUPDATE    (1u << 3)
#define APP_BATMAN_ERR_PARAM        (1u << 4)
#define APP_BATMAN_ERR_DEVICE_STATE (1u << 5)
```

Phase 1 可继续 `printf`，但必须保留全局错误标志，避免“日志里报错但业务继续误用无效数据”。

## 12. 电压采样替换

### 12.1 单体电压

建议集中定义一张映射表，避免采样、均衡、打印各写一份：

```c
typedef struct
{
    uint8_t physical_index;      // 0..5
    uint8_t bq_cell_index;       // 1,2,6,9,12,16
    uint16_t voltage_cmd;        // direct command
    uint16_t balance_mask;       // 16-bit host balance mask
} AppBatMan_CellMap_t;

static const AppBatMan_CellMap_t app_cell_map[APP_BATMAN_CELL_COUNT] =
{
    {0,  1, 0x14, 0x0001},
    {1,  2, 0x16, 0x0002},
    {2,  6, 0x1E, 0x0020},
    {3,  9, 0x24, 0x0100},
    {4, 12, 0x2A, 0x0800},
    {5, 16, 0x32, 0x8000},
};
```

采样流程：

```text
for each physical cell:
    Int_BQ76952_ReadDirect(voltage_cmd, &raw2)
    解析 little-endian signed/unsigned 2 bytes
    cell_mv[i] = value_mV
    printf("Cell%d/BQCell%d = %dmV")
```

注意：

- 不再使用旧 `gain_uv/offset_mv`。
- 不再使用旧 `cell_reg_indexes[]`。
- 电压异常时不要更新 SOC 或均衡策略。
- 第一轮上板必须用万用表逐串确认 direct command 与物理电芯对应。

### 12.2 总压、PACK、LD

| 物理量 | BQ76952 direct command | 用途 |
|---|---:|---|
| Stack Voltage | `0x34` | 电芯栈总压，SOC/显示参考 |
| PACK Pin Voltage | `0x36` | 输出端/PACK 侧检测，FET/负载状态判断 |
| LD Pin Voltage | `0x38` | 负载/预放电相关检测 |

建议：

- `bat_mv` 首选 stack voltage 或 pack voltage，要在代码注释中固定语义。
- 用 `stack_mv`、`pack_pin_mv`、`ld_pin_mv` 分别保存，避免混淆。
- `bat_mv` 若继续用于 SOC，则建议使用 stack voltage。

## 13. 电流采样与校准替换

### 13.1 硬件事实

新 BMS 板低边采样：

```text
R18 = 0.5mΩ ±1% 6W
R18.1 = GND
R18.2 = BMS_OUT-
BQ SRP/SRN 通过 100Ω + RC 接采样电阻两端
```

项目运行目标：

- 充电目标：`5A`
- 放电目标：`10A`
- 电芯最大持续放电：`15A`

### 13.2 BQ 电流读数

使用 BQ76952 direct command：

```text
CC2 Current = 0x3A
```

但以下内容必须实测：

- `CC2 Current` 在本项目 INT 层返回的是 raw、userA 还是 usermA。
- 充电时读数正负号。
- 放电时读数正负号。
- 零点偏移。
- 0.5mΩ shunt 对应的增益校准。

建议 APP 统一使用：

```c
int32_t current_ma;
// 约定：current_ma > 0 表示充电；current_ma < 0 表示放电。
```

如果 BQ 实测符号相反，则在 `App_BatMan_LoadCurrent()` 中统一转换，禁止 SOC 层再做二次反转。

### 13.3 保护目标与采样电阻关系

0.5mΩ 下 sense voltage：

| 电流 | 采样电压 |
|---:|---:|
| 5A | 2.5mV |
| 10A | 5.0mV |
| 12A | 6.0mV |
| 15A | 7.5mV |

由于 BQ OCC/OCD 阈值常按 sense voltage 档位编码，最终值必须结合 TRM 单位、BQ 校准、MOS/PCB 温升实测确定。

第一版策略建议：

- `OCC`：高于正常 5A 充电，避免 5A 正常充电误触发。
- `OCD`：高于 10A 正常放电目标，但不应直接把 15A 长期持续作为系统目标。
- `SCD`：只作为短路快速保护，必须通过限流电源/电子负载逐步验证。

## 14. 温度采样替换

### 14.1 新硬件温度资源

| BQ pin | 硬件连接 | 第一版建议 |
|---|---|---|
| `TS1` | CN4 外接热敏 + C17 2.2nF | 待确认安装位置/NTC 参数后启用 |
| `TS2` | `BMS_WAKE` | 不作为温度，作为唤醒/按键复用 |
| `TS3` | CN3 外接热敏 + C15 2.2nF | 待确认安装位置/NTC 参数后启用 |
| `DCHG` | R22 10k/B3950 NTC + C14 2.2nF | 可作为板上温度/采样电阻附近温度资源 |
| `DFETOFF` | R23 10k/B3950 NTC + C16 2.2nF | 原理图标注用于采样电阻温度检测 |
| `Int Temperature` | BQ 内部温度 | 第一版必须读取并打印 |

### 14.2 替换策略

旧项目 `App_BatMan_loadTemperature()` 读 BQ76930 TS1 ADC，并用旧 10k 分压公式换算。这在新项目中禁止迁移。

新项目分两阶段：

#### Phase 1

```text
读取 Int Temperature(0x68)
打印内部温度
暂不把 TS1/TS3/DCHG/DFETOFF 用作强保护判断
```

#### Phase 1.5

```text
确认 pin config
确认 TS1/TS3 外接 NTC 型号、安装位置
确认 R22/R23 10k/B3950 对应 pin 配置
读取 TS1/TS3/DCHG/DFETOFF direct command
确认返回单位是 0.1K 还是 mV
与实测温度对比
```

#### Phase 2

```text
把 EVE 50E 温度策略映射到 BQ 温度保护 Data Memory
并在 APP 层保留限流/禁止充放电策略
```

这个结构可以继承。

### 15.2 必须替换参数

| 参数 | 旧项目 | 新项目 |
|---|---:|---:|
| 容量 | 约 0.6Ah UPS 旧电池 | 6S1P EVE 50E：5.0Ah 典型，4.9Ah 保守 |
| 串数 | 6S | 6S |
| OCV 表 | 旧电芯表 | 待 EVE 50E 曲线替换/校准 |
| 电流方向 | `current_a > 0` 充电 | 必须由 BQ CC2 实测后统一转换 |
| 低电参考 | 旧阈值 | 2.50V 绝对截止，3.00V 寿命友好参考 |

### 15.3 第一版建议

- `App_Soc_ConfigInit()` 中容量改为 `5.0Ah` 或保守 `4.9Ah`。
- OCV 表第一版可以沿用表结构，但注释必须写“待 EVE 50E 曲线校准”。
- 电流方向未确认前，SOC 仅作为调试观察，不作为最终保护依据。
- 可用 `3.45~3.60V/cell` 作为运输 SOC `20%~30%` 的 sanity check。
- 显示 SOC 可保留旧项目 IIR 方向限制，但满电/低电修正点要按 EVE 50E 重定。

## 16. 均衡逻辑替换

### 16.1 继承旧策略

旧策略结构正确，可继承：

1. 遍历 `cell_mv[]` 找最低电压。
2. 按温度、最低电压、压差筛选候选。
3. 相邻物理电芯不能同时均衡。
4. 保留电压更高的一串。
5. 写入芯片均衡控制。
6. 读回打印。

### 16.2 新写入方式

新项目使用：

```text
CB_ACTIVE_CELLS(0x0083)
16-bit mask
0x0000 表示关闭所有 host balancing
```

物理映射：

| 物理 cell | BQ cell | mask |
|---|---|---:|
| Cell1 | Cell1 | `0x0001` |
| Cell2 | Cell2 | `0x0002` |
| Cell3 | Cell6 | `0x0020` |
| Cell4 | Cell9 | `0x0100` |
| Cell5 | Cell12 | `0x0800` |
| Cell6 | Cell16 | `0x8000` |

### 16.3 第一版业务阈值建议

| 项目 | 第一版建议 | 说明 |
|---|---:|---|
| 均衡启动电压 | `cell_mv > 4000mV` | 高 SOC 区间，保守 |
| 启动压差 | `20mV~30mV` | 待实测确认 |
| 停止压差 | `10mV~20mV` | 待实测确认 |
| 同时均衡数量 | `1` | 62Ω/1W，初次上板保守 |
| 温度范围 | 参照电芯/板温策略 | 最终结合 TS/DCHG/DFETOFF |

### 16.4 读回验证

写入后必须读回：

- `CB_ACTIVE_CELLS(0x0083)`
- `CBSTATUS1(0x0085)`
- 必要时 `CBSTATUS2/3`

上板逐路测试建议：

```text
App_BatMan_TestBalanceOneCell(physical_cell)
    physical 1 -> 0x0001
    physical 2 -> 0x0002
    physical 3 -> 0x0020
    physical 4 -> 0x0100
    physical 5 -> 0x0800
    physical 6 -> 0x8000
```

每次只开一路，观察对应 62Ω 电阻支路发热，不允许盲开多路。

## 17. 充放电 FET 控制替换

### 17.1 旧逻辑问题

旧项目 APP 直接写：

```text
BQ_SYS_CTRL2.CHG_ON
BQ_SYS_CTRL2.DSG_ON
```

这是 BQ76930 逻辑，新项目禁止继承。

### 17.2 新项目策略

保留旧 API 名称，但改语义：

```c
void App_BatMan_SetChargeState(uint8_t charge_state);
void App_BatMan_SetDischargeState(uint8_t discharge_state);
```

其中：

- `charge_state=1` 表示 APP 请求允许充电。
- `charge_state=0` 表示 APP 请求禁止充电。
- `discharge_state=1` 表示 APP 请求允许放电。
- `discharge_state=0` 表示 APP 请求禁止放电。

注意：这是 **APP 请求状态**，不保证 FET 实际导通。真实状态必须读 `FET Status(0x7F)` 以及 safety/alarm 状态。

### 17.3 推荐实现路线

第一版有两种选择：

| 路线 | 方法 | 优点 | 风险 |
|---|---|---|---|
| Subcommand 路线 | `CHG_PCHG_OFF`、`DSG_PDSG_OFF`、`ALL_FETS_ON/OFF` | 简单直观 | 读改写细节少，不如 mask 可见 |
| `FET_CONTROL` 路线 | `FET_CONTROL(0x0097)` data subcommand | 可维护 `fet_off_mask`，类似旧项目位域 | 必须确认 bit 与 `FET Status(0x7F)` 对应 |

建议 Phase 1 使用 `FET_CONTROL` + 全局 `fet_off_mask`，因为调试可见：

```text
charge_state=0    -> 设置 CHG_OFF 请求位
charge_state=1    -> 清除 CHG_OFF 请求位
discharge_state=0 -> 设置 DSG_OFF 请求位
discharge_state=1 -> 清除 DSG_OFF 请求位
写 FET_CONTROL
读 FET Status(0x7F)
读 Alarm/Safety 状态
printf request/actual
```

### 17.4 必须确认项

- `FET Options[HOST_FET_EN]` 或等效 host FET 控制使能条件。
- 是否需要先发送/确认 `FET_ENABLE` 或 manufacturing status 中 FET enable 状态。
- `FET_CONTROL` bit 与 `FET Status(0x7F)` bit 的项目 BSP 定义。
- 保护状态触发时 BQ 自动关断 CHG/DSG 的读回行为。

## 18. 保护配置替换

### 18.1 原则

电芯规格书给的是业务目标，BQ76952 Data Memory 写入值是芯片编码。两者不能混为一谈。

原则：

- APP 文档中可以写目标值。
- BSP/TRM 层写地址和单位。
- 最终写入值必须有计算过程和读回验证。
- 不继承旧 BQ76930 `PROTECT1/2/3`、`OV_TRIP`、`UV_TRIP` 公式。

### 18.2 电压保护目标

| 项目 | 业务依据 | 建议处理 |
|---|---|---|
| COV | EVE 50E 标准最高 4.20V/cell；5A 充电目标 4.10V/cell | BQ COV 作为安全保护，不简单等于 4.10V；最终阈值按充电策略和 TRM 编码确定 |
| CUV | 规格放电截止 2.50V/cell | 可作为绝对下限参考 |
| 低电策略 | 循环寿命测试使用 3.00V cutoff | APP 可用 3.00V/cell 作为寿命友好低电参考 |

建议分层：

```text
SC8815 控制正常充电 CV
APP 控制寿命友好上/下限和显示策略
BQ76952 COV/CUV 作为安全保护底线
```

### 18.3 电流保护目标

基于 0.5mΩ 采样电阻：

| 目标 | 电流 | sense voltage | 说明 |
|---|---:|---:|---|
| 正常最大充电 | 5A | 2.5mV | 不应误触发 OCC |
| 正常放电目标 | 10A | 5.0mV | 不应误触发 OCD |
| 电芯最大持续放电 | 15A | 7.5mV | 不能直接当长期系统目标 |

建议：

- OCC 目标应大于 5A 正常充电，例如先按 6~8A 级别评估。
- OCD 目标应大于 10A 工作点，但结合 MOS/PCB/电芯，先按 12~15A 范围评估。
- SCD 作为短路快速保护，不用来覆盖正常过载控制。
- 最终 code 由 TRM 单位和实际校准转换，不能只按理论电阻计算。

### 18.4 温度保护目标

来自 EVE 50E：

- 0℃ 以下禁止充电。
- 0~15℃ 充电限流到 1A。
- 15~45℃ 允许 5A。
- 45~55℃ 限流到 1A。
- 55℃ 以上禁止充电。
- -20℃ 以下禁止放电。
- 60℃ 以上禁止放电或降级。
- 70℃ 为 cell surface cut-off 硬边界。

BQ 侧温度保护最终需要：

- TS1/TS3/DCHG/DFETOFF pin config。
- NTC 分压模型。
- 保护阈值 Data Memory 地址/编码。
- 读回温度与实测温度校准。

## 19. SC8815 与 BQ76952 协同边界

| 功能 | SC8815 | BQ76952 | APP |
|---|---|---|---|
| 充电电压设定 | 负责 | 不负责正常 CV | 根据 EVE 50E 决定目标 |
| 充电电流设定 | 负责 | 可做 OCC 安全保护 | 根据温度/SOC 限流 |
| 单体过压保护 | 不负责 | 负责 COV | 策略辅助停止充电 |
| 单体欠压保护 | 不负责 | 负责 CUV | 策略辅助禁止放电 |
| 放电 FET 保护 | 不负责 | 负责 DSG | 请求/读回 |
| 电芯均衡 | 不负责 | 负责执行 | 负责策略和 mask |
| 温度限制 | 可参与充电限流 | 可做硬保护 | 策略判断 |

SC8815 的 `VBATS` 分压问题必须在上板前单独确认。若 `R17/R18` 仍为 0Ω 状态，不应启用正式充电闭环。

## 21. Bring-up 测试流程

### 21.1 上电前

1. 确认 6S 电芯接线顺序：`CELL_1-` 到 `CELL_6+`。
2. 万用表测量每串电压。
3. 确认 BMS `BAT+ / BAT- / BMS_OUT+ / BMS_OUT-` 极性。
4. 确认 SC8815 `VBATS` 分压实物状态。
5. 确认 I2C 上拉在 SYS_3V3。
6. 确认 `BMS_WAKE`、`BMS_CHIP_SHUT` 不会误触发关断。

### 21.2 通信测试

1. 检查 `BMS_CHIP_ONLINE`。
2. I2C 扫描 `0x08`。
3. 读取 DeviceNumber。
4. 检查 CRC 通信稳定性。
5. 连续读取 direct command，确认无 CRC/timeout。

### 21.3 配置测试

1. 进入 ConfigUpdate。
2. 写 `Vcell Mode=0x8923`。
3. 读回确认。
4. 写最小必要配置。
5. 退出 ConfigUpdate。
6. 重读 6 串电压。

### 21.4 电压映射测试

1. 读 6 个 cell direct command。
2. 与万用表逐串比较。
3. 确认误差范围。
4. 确认没有串位错位。

### 21.5 电流测试

1. 空载读零点。
2. 小电流充电，记录符号。
3. 小电流放电，记录符号。
4. 逐步到 1A/5A/10A，记录 BQ 读数。
5. 建立零点和增益校准。

### 21.6 FET 测试

1. 读取初始 `FET Status(0x7F)`。
2. 请求关闭 CHG，读回 FET Status。
3. 请求关闭 DSG，读回 FET Status。
4. 请求允许 CHG/DSG，确认安全状态允许时实际导通。
5. 人为触发保护前先确保限流设备。

### 21.7 均衡测试

1. 关闭所有均衡 `CB_ACTIVE_CELLS=0x0000`。
2. 单独写 Cell1 mask，确认发热支路。
3. 依次测试 Cell2/Cell3/Cell4/Cell5/Cell6。
4. 每次读回 `CB_ACTIVE_CELLS/CBSTATUS`。
5. 第一版禁止同时多路长时间均衡。

## 22. 验收标准

### 22.1 代码风格验收

- `App_BatMan.c` 能从上到下读懂 BQ 初始化、采样、SOC、均衡、FET 控制。
- APP 可以直接调用 `Int_BQ76952_*`，不强行套厚 COM wrapper。
- COM 不访问 I2C，不写 Data Memory，不控制 FET/均衡。
- INT 保留 BQ76952 协议复杂度，不为了旧风格而退化为裸寄存器读写。
- `printf` 调试信息包含：命令、原始值、换算值、读回状态、错误码。

### 22.2 硬件正确性验收

- 6S 映射为 `Cell1/2/6/9/12/16`。
- `Vcell Mode=0x8923` 写入并读回。
- I2C 地址为 `0x08 / 0x10 / 0x11`。
- 默认 CRC 通信稳定。
- 均衡 mask 是 16-bit BQ mask，不是 6-bit physical mask。
- 不存在旧 `BQ_CELLBAL`、`BQ_SYS_CTRL`、`BQ_PROTECT`、`BQ769_RegisterGroup`。

### 22.3 Bring-up 功能验收

- DeviceNumber 可读。
- 6 串电压与万用表匹配。
- Stack/PACK/LD 电压可读且意义清楚。
- CC2 电流符号、零点、增益完成记录。
- Int temperature 可读。
- TS/DCHG/DFETOFF 资源完成至少一轮温度趋势验证。
- FET request 与 actual status 均打印。
- 单路均衡实际发热通道与物理 cell 对应。
- SC8815 不在 VBATS 分压异常状态下闭环充电。

## 24. Phase 2 收敛建议

Phase 1 调通后，可以考虑：

1. 把 `FET_CONTROL`、`CB_ACTIVE_CELLS`、保护状态读回封装到薄 BQIF 层。
2. 把 Data Memory 配置做成表驱动，并保留读回校验。
3. 把 `printf` 切到可开关日志宏。
4. 给所有采样函数增加返回状态码。
5. 用 EVE 50E 实测 OCV-SOC 曲线替换旧表。
6. 完成温度模型后，把 TS/DCHG/DFETOFF 温度纳入保护与限流。
7. 固化电流校准参数。
8. 将 bring-up 测试函数用宏隔离，避免量产误调用。
