# SOC 参数整定教程

这份文档不是讲原理，而是讲“怎么一步一步把参数调到能用”。

适用对象：当前项目里的 `App_Soc.c` / `App_Soc.h`

需要重点调的 5 个参数是：

1. `capacity_ah`
2. `r0_ohm`
3. `p_sigma_soc_per_s`（教程里有时写成 `q_soc_per_s`，本项目代码名是这个）
4. `r_sigma_soc`（教程里有时写成 `sigma_soc`，本项目代码名是这个）
5. `max_update_current_a`

先记一句最实用的话：

- `capacity_ah` 决定“积分会不会越跑越偏”
- `r0_ohm` 决定“电压补偿准不准”
- `p_sigma_soc_per_s` 决定“愿不愿意承认积分会漂”
- `r_sigma_soc` 决定“愿不愿意相信电压查表”
- `max_update_current_a` 决定“什么时候允许用电压修正 SOC”

---

## 一、先理解整定顺序

不要 5 个参数一起乱改，正确顺序是：

1. 先定 `capacity_ah`
2. 再定 `r0_ohm`
3. 再定 `max_update_current_a`
4. 最后微调 `p_sigma_soc_per_s`
5. 最后微调 `r_sigma_soc`

原因很简单：

- 容量不准，积分一定会长期漂
- 内阻不准，电压补偿一定错
- 低电流阈值不准，滤波器会在“不该修正的时候乱修正”
- 前面都差不多以后，再调两个“信谁更多”的滤波参数才有意义

---

## 二、推荐的起始值

如果你现在只想先跑起来，先用下面这组初值：

```c
capacity_ah = 电池标称容量
r0_ohm = 0.05f
p_sigma_soc_per_s = 1e-4f
r_sigma_soc = 0.03f
max_update_current_a = 0.10f
```

如果系统静置时电流噪声比较大，可以先这样：

```c
max_update_current_a = 0.15f ~ 0.20f
```

如果电流采样很准、系统很稳，也可以先这样：

```c
p_sigma_soc_per_s = 1e-5f
```

---

## 三、整定前准备

开始之前，先准备好下面几个条件：

1. 能看到这几个量：`SOC`、`pack_mv`、`current_a`
2. 知道电池包串数，本项目是 `cell_num = 6`
3. 能做两类测试：
   - 小电流静置测试
   - 有明显负载变化的充放电测试
4. 最好能记录一段日志，至少包含：
   - 时间
   - 电池包电压
   - 电流
   - SOC

如果没有日志工具，最少也要能串口打印，肉眼看参数变化也能调，只是慢一点。

---

## 四、一步一步整定

## 第 1 步：先把 `capacity_ah` 调准

### 它是干什么的

这个参数直接参与安时积分：

```c
soc_delta = current_a * dt / (capacity_ah * 3600)
```

它不准，SOC 会“越跑越偏”，而且是长期偏。

### 怎么开始填

先直接填电池铭牌标称值。

例子：

- 0.6Ah 就填 `0.6f`
- 7Ah 就填 `7.0f`
- 20Ah 就填 `20.0f`

### 怎么验证

做一次尽量完整的充放电验证：

1. 让电池先充满
2. 静置一会，等电压比较稳定
3. 记下此时 SOC 应该接近 100%
4. 用相对稳定的电流放电
5. 放到你的系统截止点
6. 看中途和结束时 SOC 是否明显偏大或偏小

### 怎么改

如果出现下面现象，按这个方向改：

- 实际电池已经快没电了，但 SOC 还显示很多
  - 说明 `capacity_ah` 填大了
  - 把它调小

- 实际电池还有不少电，但 SOC 掉得太快
  - 说明 `capacity_ah` 填小了
  - 把它调大

### 建议改法

不要一次改太猛，每次改 `5% ~ 10%`。

例子：

- 当前是 `0.60`
- 如果发现 SOC 普遍偏高，就先改成 `0.55`
- 如果发现 SOC 普遍偏低，就先改成 `0.65`

### 什么时候算调好了

满足下面两点就可以先认为合格：

1. 中段 SOC 下降速度基本符合体感
2. 一次完整放电结束时，SOC 和实际剩余电量大体一致

---

## 第 2 步：再把 `r0_ohm` 调准

### 它是干什么的

代码里会先做一个电压补偿：

```c
v_ocv_pack_mv = pack_mv - current_a * r0_ohm * 1000
```

意思是：把负载压降补回去，让端电压更接近 OCV。

如果这个值不准，后面的“电压修正 SOC”就会被带偏。

### 最简单的测法

找一个中间 SOC，比如 40% 到 70%，做一次电流阶跃测试：

1. 让电池先静置
2. 记下静置电压
3. 突然加上一个较稳定负载
4. 记录电流变化量 `ΔI`
5. 记录电压瞬间变化量 `ΔV`
6. 粗估：

```text
r0_ohm ≈ ΔV / ΔI
```

注意单位统一：

- `ΔV` 用伏特 V
- `ΔI` 用安培 A

例子：

- 电流从 `0A` 变到 `-2A`
- 电压瞬间下降 `0.12V`
- 那么 `r0_ohm ≈ 0.12 / 2 = 0.06Ω`

### 没条件测怎么办

可以直接从 `0.05f` 开始，再根据现象修。

### 怎么看方向对不对

看“有负载时的电压修正结果”是否合理。

- `r0_ohm` 太小：
  - 放电时补偿不够
  - 算出来的 OCV 偏低
  - 滤波器容易把 SOC 往低了拉

- `r0_ohm` 太大：
  - 放电时补偿过头
  - 算出来的 OCV 偏高
  - 滤波器容易把 SOC 往高了拉

### 实操判断法

做一次“放电一段时间后停负载静置”的测试：

1. 先让系统带负载运行
2. 观察此时 SOC
3. 然后去掉负载，静置
4. 看静置后 SOC 的修正方向

如果经常出现：

- 一静置，SOC 就明显往上跳很多
  - 通常说明放电时电压被压低得太多，而你的补偿不够
  - 优先怀疑 `r0_ohm` 偏小

- 一静置，SOC 反而往下掉，且和体感不符
  - 优先怀疑 `r0_ohm` 偏大

### 建议改法

每次改 `0.005 ~ 0.01Ω`，不要一次改太多。

---

## 第 3 步：把 `max_update_current_a` 设到合适

### 它是干什么的

只有满足：

```c
fabs(current_a) <= max_update_current_a
```

系统才允许用电压观测来修正 SOC。

所以它本质上是在回答一个问题：

“电流小到什么程度，端电压才勉强可以当 OCV 看？”

### 最傻瓜的设定方法

先观察系统静置时的电流抖动。

比如静置时电流在下面范围内乱跳：

- `-0.03A ~ 0.04A`
- `-0.06A ~ 0.05A`

那就把阈值设成“略大于静置噪声上限”的值。

参考经验：

- 静置噪声在 `0.03A` 左右：先设 `0.08A ~ 0.10A`
- 静置噪声在 `0.05A` 左右：先设 `0.10A ~ 0.15A`
- 静置噪声在 `0.10A` 左右：先设 `0.15A ~ 0.20A`

### 怎么看是不是太小

如果太小，会出现：

- 明明已经停机静置了，但很久都不做电压修正
- SOC 长时间只按积分慢慢走
- 静置后纠偏很慢

这时就把 `max_update_current_a` 调大一点。

### 怎么看是不是太大

如果太大，会出现：

- 负载还在变化时，系统也在频繁用电压修正
- SOC 在带载过程中乱跳
- 一会儿被拉高，一会儿被拉低

这时就把 `max_update_current_a` 调小一点。

### 建议改法

每次改 `0.02A ~ 0.05A`。

---

## 第 4 步：再调 `p_sigma_soc_per_s`

### 它是干什么的

它控制“预测误差累积速度”。

可以简单理解成：

- 值大：系统觉得“积分不一定靠谱”，所以更愿意让电压来纠偏
- 值小：系统觉得“积分挺靠谱”，所以更不愿意被电压拉着走

### 起始值建议

先用：

```c
p_sigma_soc_per_s = 1e-4f
```

如果你对电流采样很有信心，也可以从 `1e-5f` 开始。

### 怎么看是不是太小

如果太小，会出现：

- SOC 非常稳，但明显“认死理”
- 静置很久以后，电压已经表明 SOC 应该修正，但它改得很慢
- 长时间运行后，积分偏差不容易被拉回来

这时把它调大。

### 怎么看是不是太大

如果太大，会出现：

- 只要一进入可观测区，SOC 就容易被电压带着跑
- SOC 修正过快，甚至有点抖
- 电压稍有波动，SOC 就变得敏感

这时把它调小。

### 建议改法

按 10 倍级别调最省事：

- 太保守：`1e-5 -> 1e-4 -> 1e-3`
- 太激进：`1e-3 -> 1e-4 -> 1e-5`

通常先在这几个数量级里找，不要一点点抠小数。

---

## 第 5 步：最后调 `r_sigma_soc`

### 它是干什么的

它表示“电压查表得到的 SOC 测量值到底有多可信”。

可以简单理解成：

- 值小：更相信电压
- 值大：更相信积分

### 起始值建议

先用：

```c
r_sigma_soc = 0.03f
```

也就是把电压查表 SOC 当成“误差大约 3%”的观测值。

### 怎么看是不是太小

如果太小，会出现：

- 一进入静置区，SOC 修正太猛
- 在 OCV 平台区，SOC 可能抖来抖去
- 电压一点小变化，也能引起明显 SOC 波动

这时把它调大，比如：

```c
0.02 -> 0.03 -> 0.05
```

### 怎么看是不是太大

如果太大，会出现：

- 明明静置很久了，SOC 还是不太愿意修正
- 电压查表已经能看出偏差，但滤波器几乎没动作

这时把它调小，比如：

```c
0.05 -> 0.03 -> 0.02
```

### 最实用的判断标准

静置 5 到 10 分钟后：

- 如果 SOC 仍然几乎不动，偏保守了
- 如果 SOC 一下跳很多，偏激进了
- 如果是缓慢、稳定地往合理值靠近，说明差不多

---

## 五、推荐你这样整定，最省时间

如果你只想尽快调出“能用版本”，按下面做：

1. `capacity_ah` 先填标称值
2. `r0_ohm` 先填 `0.05`
3. `max_update_current_a` 先填 `0.10`
4. `p_sigma_soc_per_s` 先填 `1e-4`
5. `r_sigma_soc` 先填 `0.03`
6. 跑一次“带载 -> 停载静置”测试
7. 先看静置后 SOC 是不是乱跳
8. 如果乱跳，先减小 `max_update_current_a`，再增大 `r_sigma_soc`
9. 如果静置后几乎不纠偏，先增大 `max_update_current_a`，再增大 `p_sigma_soc_per_s` 或减小 `r_sigma_soc`
10. 如果长期越跑越偏，回头重新修 `capacity_ah`
11. 如果一带负载就明显估低或估高，回头重新修 `r0_ohm`

记住一个原则：

- 先修“模型参数”：`capacity_ah`、`r0_ohm`
- 再修“滤波权重”：`p_sigma_soc_per_s`、`r_sigma_soc`

---

## 六、看到什么现象，优先改哪个参数

### 现象 1：SOC 长期越跑越偏

优先检查：

1. `capacity_ah`
2. 电流采样是否有零漂
3. `p_sigma_soc_per_s` 是否太小

### 现象 2：静置后 SOC 不愿意修正

优先检查：

1. `max_update_current_a` 是否太小
2. `p_sigma_soc_per_s` 是否太小
3. `r_sigma_soc` 是否太大

### 现象 3：静置后 SOC 一跳一大截

优先检查：

1. `r_sigma_soc` 是否太小
2. `max_update_current_a` 是否太大
3. `r0_ohm` 是否不准

### 现象 4：带载过程中 SOC 乱跳

优先检查：

1. `max_update_current_a` 是否太大
2. `r_sigma_soc` 是否太小
3. 电压采样是否抖动太大

### 现象 5：放电时 SOC 总偏高或偏低

优先检查：

1. `r0_ohm`
2. `capacity_ah`

---

## 七、一个可直接抄的初版参数

如果你现在的电池包比较小，系统也不是大电流冲击型，可以先用：

```c
cfg->capacity_ah = 按你的电池标称容量填写;
cfg->r0_ohm = 0.05f;
cfg->p_sigma_soc_per_s = 1e-4f;
cfg->r_sigma_soc = 0.03f;
cfg->max_update_current_a = 0.10f;
```

然后按下面顺序微调：

1. 先只改 `capacity_ah`
2. 再只改 `r0_ohm`
3. 再只改 `max_update_current_a`
4. 最后才动 `p_sigma_soc_per_s` 和 `r_sigma_soc`

每次只改 1 个参数，否则你会很难判断到底是谁起作用。

---

## 八、最后给一个最短版结论

如果你懒得看全文，就记下面这几句：

1. `capacity_ah` 不准，SOC 会长期漂
2. `r0_ohm` 不准，静置修正方向容易错
3. `max_update_current_a` 太大，带载时会乱修正
4. `p_sigma_soc_per_s` 太小，SOC 会很死，不愿纠偏
5. `r_sigma_soc` 太小，SOC 会太相信电压，容易跳

推荐整定顺序永远是：

```text
capacity_ah -> r0_ohm -> max_update_current_a -> p_sigma_soc_per_s -> r_sigma_soc
```
